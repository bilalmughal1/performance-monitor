# Contributing to Web Performance Monitor

First off, thank you for considering contributing to this project! Your help is greatly appreciated.

## Where to start?

If you're looking for a place to start, check out the [open issues](https://github.com/bilalmughal1/performance-monitor/issues). You can also contribute by improving the documentation, adding tests, or proposing new features.

## How to contribute?

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** to your local machine:
    `git clone https://github.com/YOUR_USERNAME/performance-monitor.git`
3.  **Create a new branch** for your changes:
    `git checkout -b your-branch-name`
4.  **Make your changes**.
5.  **Commit your changes** with a descriptive commit message:
    `git commit -m "Your commit message"`
6.  **Push your changes** to your fork:
    `git push origin your-branch-name`
7.  **Create a pull request** from your fork to the main repository.

## Development setup

1.  **Install dependencies**:
    `npm install`
2.  **Set up environment variables**:
    Create a `.env.local` file and add the necessary environment variables. You can use `env.example` as a template.
3.  **Run the development server**:
    `npm run dev`

## Coding guidelines

*   **Follow the existing code style**.
*   **Write clear and concise code**.
*   **Add comments** where necessary.
*   **Write tests** for your changes.

## Submitting a pull request

Before submitting a pull request, please make sure that:

*   You have **rebased your branch** on the latest version of the `main` branch.
*   You have **run the tests** and they all pass.
*   You have **updated the documentation** if necessary.
*   Your pull request has a **clear and descriptive title and description**.

Thank you for your contribution!
